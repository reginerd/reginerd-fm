Run Your Own AI Radio Station

A complete walkthrough for setting up a 24/7 AI-powered internet radio station using the RGNRD-FM codebase.

2 Clone & Setup

# Clone the repo
git clone https://github.com/keltokhy/writ-fm.git
cd writ-fm

# Install Python dependencies
uv sync

# Copy example configs
cp config/icecast.xml.example config/icecast.xml
cp mac/config.yaml.example mac/config.yaml

# Make the writ CLI executable
chmod +x writ run_operator.sh

3 Set Up TTS

RGNRD-FM uses Kokoro TTS — fast, CPU-friendly, with 28 built-in voices (American & British). Each show in config/schedule.yaml is assigned a voice ID; hosts have distinctive deliveries that survive long-form segments.

Install Kokoro

  • ~200 MB model download (first run)
  • CPU-only, runs on anything
  • Apple Silicon recommended for parallel generation
cd mac/kokoro
uv venv && uv pip install kokoro soundfile

4 Configure

config/icecast.xml

Set your passwords. The source-password is what the streamer uses to connect:

<authentication>
    <source-password>your_source_password</source-password>
    <admin-user>admin</admin-user>
    <admin-password>your_admin_password</admin-password>
</authentication>

mac/config.yaml

Main station configuration. Key settings:

  • icecast_password — must match source-password above
  • icecast_host, icecast_port, icecast_mount — defaults are fine for localhost

config/schedule.yaml

Defines your shows, hosts, voices, and the daily/weekly schedule. Each show has:

  • name, description — identity for the LLM prompt
  • host — persona name from persona.py
  • topic_focus — what the AI talks about
  • bumper_style — music genre for AI-generated bumpers
  • voices — TTS voice IDs for host and optional guest

5 Music Bumpers (Optional)

RGNRD-FM is talk-first — long-form talk segments dominate, with short music bumpers (1–2 tracks) between them. Bumpers are AI-generated via music-gen.server (ACE-Step model). Each show has a bumper_style in config/schedule.yaml — ambient, jazz, downtempo, soul, indie.

To run the music-gen server alongside RGNRD-FM:

# Clone music-gen.server next to writ-fm
cd ..
git clone https://github.com/kortexa-ai/music-gen.server
cd writ-fm

# Start it via the writ helper (uses tmux)
bash mac/start_music_gen.sh server

Skip this step if you want to start without music — the operator handles a missing music-gen gracefully (talk segments still generate).

6 Generate Initial Content

Before your first broadcast, stock at least the current and next slot with talk segments. The writ CLI wraps the generators:

# Stock 3 talk segments for the currently-airing slot (~30 min content)
./writ generate talk

# Stock segments for a specific show
./writ generate talk --show midnight_signal

# Stock music bumpers (requires music-gen.server)
./writ generate music

# See what's stocked across upcoming slots
./writ generate status

Or call the generators directly for finer control:

# Stock the next 4 airings to a minimum of 6 segments each
uv run python mac/content_generator/talk_generator.py --stock-ahead 4 --min 6

# Stock all bumper pools to 5+
uv run python mac/content_generator/music_bumper_generator.py --all --min 5

Talk segments land in output/talk_segments/{show_id}/{slot}/ — each airing gets its own slot folder, and content there only plays during that specific airing before being archived.

7 Start Streaming

The writ CLI manages all components via tmux:

./writ start          # Start core streaming stack (icecast, stream, tunnel)
./writ start all      # Start core + content daemons
./writ status         # Health check all components
./writ stop           # Stop everything

Or start components individually, in order:

./writ start icecast  # Icecast streaming server (:8000)
./writ start stream   # ezstream + feeder.py + API (:8001)
./writ start tunnel   # cloudflared tunnel (optional)
./writ start content  # music-gen, operator, listener

Listen at http://localhost:8000/stream. The now-playing API runs at http://localhost:8001/now-playing as a daemon thread inside the streamer.

8 Run 24/7

Two daemons keep the station alive without manual intervention:

Operator daemon (content + health)

Runs Claude Code on a 15-minute loop. Each pass: health-checks the stream, stocks talk and music for upcoming slots, processes listener messages, and leaves an editorial note in the station ledger so future runs can pick up threads instead of repeating themselves.

./writ start operator             # Start the daemon (tmux-managed)
./writ logs operator -f           # Follow live
./writ attach operator            # Attach to the tmux window

Listener daemon (message responses)

Polls ~/.writ/messages.json for unread listener messages and generates on-air responses through the appropriate host's voice.

./writ start listener

Run a single operator pass manually

./run_operator.sh                 # One pass, with lock + 30-min timeout

Note on cron and launchd (macOS): If your repo lives on an external /Volumes/ drive, both cron and user LaunchAgents will fail with “Operation not permitted” (TCC blocks daemon-spawned bash from accessing external volumes without Full Disk Access). Run the operator from a tmux session inherited from your Terminal (the default ./writ start operator path) or move the project to ~/. Linux users can use cron freely.

9 Make it Public

Your stream is at localhost:8000/stream. To let the world listen:

Option A: Cloudflare Tunnel (simplest)

# Install cloudflared
brew install cloudflared  # or apt/snap

# Authenticate
cloudflared tunnel login

# Create a tunnel
cloudflared tunnel create writ-radio

# Route your domain
cloudflared tunnel route dns writ-radio radio.yourdomain.com

# Run it
cloudflared tunnel run writ-radio

Option B: Reverse proxy (nginx)

server {
    listen 443 ssl;
    server_name radio.yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_buffering off;  # important for streaming
    }
}

10 Customize

Change the station persona

Edit mac/content_generator/persona.py to redefine your station's identity — name, voice, speaking style, philosophical leanings. This is the system prompt the LLM uses for all DJ scripts.

Modify the schedule

Edit config/schedule.yaml to add, remove, or rearrange shows. You can set different voices, topic focuses, and bumper styles per show. The overrides section handles weekly specials.

Add or change voices

Kokoro ships with 28 voices — check mac/kokoro/tts.py for the full list. Assign them per-show in schedule.yaml under the voices key.

Each show in config/schedule.yaml can also assign a different voice for a guest host (used by panel-format shows like Crosswire).
